Payroll Exception Report - Additional Requirements

As a payroll user I want some changes to the payroll exception report (see US7429) so that some exceptions that weren't specified correctly are amended and become meaningful.


Many changes were made to the exception report on US7429 the following were not specified but picked up in the acceptance review of US7429 and need to be addressed:
  • 1.       – item 11. The label is currently “Employees whose salary is maximum” this has not changed and was not specified to be changed. However I noticed it doesn’t make much sense and should really read “Employees whose salary is over maximum”. We are not amending this before the deployment. Label should be changed
  • 2.        – item 12. Where an employee has both salary and hourly payments. The employee appears on the report with the message to clearly indicate why, however the values in the maximum and unit columns aren’t very meaningful and it’s quite hard to make them so. Therefore we will not address this before deployment. Values should be removed
  • 3.       - Item 15 the check for statutory payments – this doesn’t include ShPP. It wasn’t specified in this US and therefore won’t be addressed before deployment. ShPP needs to be included and ASPP removed
  • 4.       - Item 21 tax rebate check – this works only on employees with tax code with a suffix of L. This was previously agreed with you and therefore this won’t be changed before deployment. It should be changed to just add an employee that has a tax rebate.
